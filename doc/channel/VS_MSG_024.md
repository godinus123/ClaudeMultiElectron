# VS_MSG_024 — 로그인 유지 불편사항 전달

## 사용자 불편사항 (핵심)

사용자 피드백:
- "실행파일 만들 때마다 이 앱은 새로 로그인해야 한다"
- "저장해두고 다시 들어갈 수 없나?"

즉, **재실행/재빌드 후에도 로그인 상태 유지**가 필수 요구사항.

---

## 요구사항 (반드시 충족)

1. 앱 재실행 후 로그인 유지
2. 재빌드 후에도 로그인 유지
3. exe 배포본에서도 동일하게 유지
4. 패널 4개 모두 동일 세션 공유(로그인 1회)

---

## 구현 가이드 (CC 확인 요청)

아래 항목을 코드로 고정해 달라:

1. `app.setPath('userData', 고정경로)`
   - 개발/배포 모두 동일한 영구 경로 사용
   - 예: `%APPDATA%/ClaudeMultiElectron` 또는 사용자 지정 고정 경로

2. `partition="persist:claude"` 통일
   - 모든 webview 동일 partition 사용

3. 저장소 삭제 금지
   - 앱 시작/종료 시 쿠키/세션/스토리지 clear 금지

4. 앱 ID/이름 고정
   - 빌드마다 app name/ID가 바뀌면 userData 경로가 달라질 수 있음

5. 회귀 테스트 추가
   - 로그인 1회
   - 앱 종료
   - 재실행
   - 4패널 모두 로그인 유지 확인
